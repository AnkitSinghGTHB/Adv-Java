### 1. Graphical User Interface (GUI) Concept
A Graphical User Interface (GUI) is a visual way for users to interact with an application. Instead of typing text-based commands into a terminal, users interact with graphical icons, buttons, text fields, checkboxes, and windows. It makes applications drastically more intuitive and user-friendly.

### 2. Main Features of GUI
*   **Visual Elements:** Employs standard visual metaphors like windows, icons, menus, and pointers (WIMP).
*   **Event-Driven:** The execution flow of a GUI program is determined by user actions (clicking a button, typing text).
*   **Stateful:** The interface maintains state visually (e.g., a checkbox remains visibly checked until clicked again).

### 3. Technologies Used in Java for GUI
Java provides major toolkits to build GUIs:
1.  **AWT (Abstract Window Toolkit):** The original Java GUI toolkit. It is heavyweight, meaning it relies heavily on the underlying operating system's native GUI peers, making AWT applications look different across different operating systems.
2.  **Swing:** A massive improvement over AWT. It is lightweight, writing GUI components entirely in Java without connecting to OS-native peers, ensuring a perfectly uniform "look-and-feel" across all platforms.
3.  **JavaFX:** The modern, hardware-accelerated GUI toolkit designed to replace Swing. It supports CSS for styling and XML (FXML) for layout structures.

### 4. Role of Java Foundation Classes (JFC)
The **Java Foundation Classes (JFC)** is a comprehensive set of GUI features to help developers build sophisticated graphical applications. 
*   **Function:** JFC acts as the umbrella framework that encompasses Swing components, 2D API for advanced drawing/rendering, Accessibility API, and Pluggable Look-and-Feel support.
*   **Role:** By utilizing JFC, developers can easily inject robust GUI components, support drag-and-drop mechanisms, and create applications that are accessible to users with disabilities, all while maintaining perfect platform independence.
