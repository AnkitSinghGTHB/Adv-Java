### 1. Java Architecture (JDK, JRE, JVM)
Java's architecture is a combination of compilation and interpretation, making it highly versatile. It consists of three primary components: JDK, JRE, and JVM.

*   **JDK (Java Development Kit):** The complete software development environment used to develop Java applications. It physically exists and contains the JRE + development tools (like the compiler `javac`, archiver `jar`, documentation generator `javadoc`, etc.).
*   **JRE (Java Runtime Environment):** A software package that provides the minimum requirements for executing a Java application. It physically exists and contains the JVM + core classes and supporting libraries.
*   **JVM (Java Virtual Machine):** An abstract machine that provides the runtime environment in which Java bytecode can be executed. It is platform-dependent (there's a different JVM for Windows, Linux, Mac).

### 2. Platform Independence
Java achieves platform independence through its **"Write Once, Run Anywhere" (WORA)** paradigm.
1.  **Compilation:** When a Java program (`.java` file) is compiled, the `javac` compiler does not generate native machine code. Instead, it generates an intermediate, CPU-independent code called **Bytecode** (`.class` file).
2.  **Execution (Interpretation/JIT):** This bytecode can be sent to any platform. As long as the target platform has a JVM installed, the JVM will read the generic bytecode and translate it into the specific native machine code of that underlying operating system.
